## 执行摘要

### 项目概述

智能求职助手是一个基于人工智能的求职辅助平台，旨在帮助求职者优化求职流程。该系统利用先进的AI技术提供智能简历解析、个性化职位推荐、对话式简历优化以及全面的申请跟踪功能。

### 核心价值主张

- **节省时间**: 自动简历解析减少手动数据录入
- **智能匹配**: 基于语义理解的职位推荐
- **个性化指导**: AI驱动的简历优化建议
- **流程管理**: 集中化的申请跟踪和面试安排

### 目标用户群体

| 用户类型  | 描述       | 主要需求      |
|-------|----------|-----------|
| 应届毕业生 | 首次进入就业市场 | 简历构建、职位发现 |
| 转行者   | 切换职业领域   | 技能匹配、简历重塑 |
| 在职求职者 | 寻求更好机会   | 高效申请、面试管理 |

### 关键成功因素

1. 准确的简历解析（F1分数 > 0.85）
2. 相关的职位推荐（NDCG@5 > 0.8）
3. 响应迅速的用户界面（p95延迟 < 10秒）
4. 高用户满意度（对话评分 > 4.0）

## 项目范围

### 范围内

#### 核心功能

1. **用户注册与认证**
    - JWT-based用户认证系统
    - 安全的用户会话管理
    - 用户资料管理

2. **简历上传与解析**
    - 支持PDF和Word文档格式
    - 异步预处理管道
    - 结构化信息提取
    - 简历摘要生成（2-5KB）

3. **职位智能匹配**
    - 基于向量的语义搜索
    - 匹配分数计算
    - 个性化职位推荐
    - 分页结果展示

4. **对话式简历优化**
    - 聊天界面交互
    - AI驱动的优化建议
    - 对话历史管理
    - 上下文感知响应

5. **申请跟踪系统**
    - 职位申请记录
    - 面试日程管理
    - 申请状态跟踪
    - 历史记录查询

#### 技术实现

1. **系统架构**
    - 单体仓库结构（Monorepo）
    - 微服务通信
    - 消息队列集成
    - Docker Compose部署

2. **AI集成**
    - 向量搜索与嵌入
    - 结构化输出解析
    - LLM API集成
    - 内存/对话管理
    - RAG（检索增强生成）

3. **数据管理**
    - PostgreSQL业务数据存储
    - pgvector向量数据存储
    - 统一存储架构
    - 数据隔离实现

### 范围外

| 项目     | 说明              | 未来考虑   |
|--------|-----------------|--------|
| 企业端功能  | 雇主职位发布和管理       | 第二阶段   |
| 实时通知   | 推送通知系统          | 第二阶段   |
| 移动端应用  | 原生iOS/Android应用 | 第三阶段   |
| 多语言支持  | 除中文外的其他语言       | 根据需求评估 |
| 薪资分析   | 市场薪资数据          | 未来增强   |
| 社交网络集成 | LinkedIn等平台连接   | 未来增强   |
| 视频面试   | 内置视频面试功能        | 不在路线图中 |
| 简历模板   | 可视化简历构建器        | 未来增强   |

## 系统约束

### 技术约束

| 约束           | 描述                                             | 影响                |
|--------------|------------------------------------------------|-------------------|
| **技术栈固定**    | 必须使用React + Spring Boot + FastAPI + PostgreSQL | 限制技术选择灵活性         |
| **AI服务隔离**   | AI服务不能直接访问数据库                                  | 需要消息队列通信          |
| **统一存储**     | 必须使用PostgreSQL + pgvector                      | 不能使用专用向量数据库       |
| **Docker部署** | 必须使用Docker Compose                             | 限制Kubernetes等编排工具 |
| **开源优先**     | 优先使用开源组件                                       | 限制商业软件使用          |

### 业务约束

| 约束       | 描述            | 影响        |
|----------|---------------|-----------|
| **开发周期** | 25天开发周期       | 限制功能范围    |
| **团队规模** | 小型开发团队        | 限制并行开发能力  |
| **预算限制** | 有限的AI API调用预算 | 需要成本控制和优化 |
| **合规要求** | 遵守数据保护法规      | 需要数据安全措施  |

### 集成约束

| 约束          | 描述               | 影响               |
|-------------|------------------|------------------|
| **LLM API** | 依赖外部LLM服务        | 需要故障转移机制         |

| **向量计算**    | 依赖pgvector扩展     | 需要PostgreSQL特定版本 |
