## 執行摘要

### 專案概述

JobCopilot 是一個使用 AI 幫助求職者改進履歷、匹配職位、追蹤申請進度的平台。系統透過語義向量匹配理解履歷與職位要求的關聯，並提供對話式履歷最佳化和申請管理功能。

### 核心價值

- **節省時間**: 自動履歷剖析減少手動資料輸入
- **語義配對**: 基於語意理解的職位推薦
- **履歷最佳化**: AI 輔助的履歷最佳化建議
- **流程管理**: 集中化的申請追蹤和面試安排

### 目標使用者群體

| 使用者類型  | 描述       | 主要需求      |
|-------|----------|-----------|
| 應屆畢業生 | 首次進入就業市場 | 履歷建構、職位發現 |
| 轉行者   | 切換職業領域   | 技能配對、履歷重塑 |
| 在職求職者 | 尋求更好機會   | 高效申請、面試管理 |

### 關鍵成功因素

1. 準確的履歷剖析（F1 分數 > 0.85）
2. 相關的職位推薦（NDCG@5 > 0.8）
3. 反應迅速的使用者介面（p95 延遲 < 10 秒）
4. 高使用者滿意度（對話評分 > 4.0）

## 專案範圍

### 範圍內

#### 核心功能

1. **使用者註冊與認證**
    - JWT-based 使用者認證系統
    - 安全的使用者會話管理
    - 使用者資料管理

2. **履歷上傳與剖析**
    - 支援 PDF 和 Word 檔案格式
    - 非同步預處理管道
    - 結構化資訊提取
    - 履歷摘要生成（2-5KB）

3. **職位智能配對**
    - 基於向量的語意搜尋
    - 配對分數計算
    - 個人化職位推薦
    - 分頁結果展示

4. **對話式履歷最佳化**
    - 聊天介面互動
    - AI 輔助的最佳化建議
    - 對話歷史管理
    - 上下文感知回應

5. **申請追蹤系統**
    - 職位申請記錄
    - 面試日程管理
    - 申請狀態追蹤
    - 歷史記錄查詢

#### 技術實現

1. **系統架構**
    - 單體倉庫結構（Monorepo）
    - 微服務通訊
    - 訊息佇列整合
    - Docker Compose 部署

2. **AI 整合**
    - 向量搜尋與嵌入
    - 結構化輸出剖析
    - LLM API 整合
    - 記憶體/對話管理
    - RAG（檢索增強生成）

3. **資料管理**
    - PostgreSQL 業務資料儲存
    - pgvector 向量資料儲存
    - 統一儲存架構
    - 資料隔離實現

## 系統限制

### 技術限制

| 限制           | 描述                                             | 影響                |
|--------------|------------------------------------------------|-------------------|
| **技術棧固定**    | 必須使用 React + Spring Boot + FastAPI + PostgreSQL | 限制技術選擇彈性         |
| **AI 服務隔離**   | AI 服務不能直接存取資料庫                                  | 需要訊息佇列通訊          |
| **統一儲存**     | 必須使用 PostgreSQL + pgvector                      | 不能使用專用向量資料庫       |
| **Docker 部署** | 必須使用 Docker Compose                             | 限制 Kubernetes 等編排工具 |
| **開源優先**     | 優先使用開源元件                                       | 限制商業軟體使用          |

### 業務限制

| 限制       | 描述            | 影響        |
|----------|---------------|-----------|
| **開發週期** | 25 天開發週期       | 限制功能範圍    |
| **團隊規模** | 小型開發團隊        | 限制並行開發能力  |
| **預算限制** | 有限的 AI API 呼叫預算 | 需要成本控制和最佳化 |
| **合規要求** | 遵守資料保護法規      | 需要資料安全措施  |

### 整合限制

| 限制          | 描述               | 影響               |
|-------------|------------------|------------------|
| **LLM API** | 相依外部 LLM 服務        | 需要故障轉移機制         |
| **檔案處理**    | 相依第三方 PDF/Word 剖析庫 | 需要處理剖析失敗         |
| **向量計算**    | 相依 pgvector 擴充     | 需要 PostgreSQL 特定版本 |
